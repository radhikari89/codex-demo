# Architecture Knowledge

Status: Living knowledge base

This folder stores reusable architecture concepts, provider notes, tradeoff summaries, and learning references that are not themselves decisions, drafts, workflows, or diagrams.

Use this area for notes that should remain useful across multiple stories, ADRs, and prototypes.

## Topics

- [Security and Auth](security-and-auth/README.md)

## Suggested Structure

- `security-and-auth/`: authentication, authorization, identity providers, tokens, sessions, and security concepts.
- `cloud-and-deployment/`: AWS, hosting, networking, DNS, CDN, environments, and operational notes.
- `data-and-persistence/`: database, schema migration, data ownership, and persistence concepts.
- `frontend-and-ux/`: frontend architecture, routing, state, accessibility, and interaction concepts.
- `ai-and-automation/`: AI agents, automation patterns, evaluation, and workflow notes.

Create a new topic folder when a concept does not naturally fit an existing category.
